# Product Specification

## 1. Product Overview

### Product Name
John Dave Decena Portfolio

### Product Type
Single-page personal portfolio website

### Primary Goal
Build a modern, premium portfolio website that supports:
- full-time remote job applications
- freelance or part-time client acquisition
- stronger GitHub and public engineering credibility

### Primary Audience Priority
1. Technical interviewers
2. Engineering managers
3. Recruiters / HR
4. Freelance clients

## 2. Product Positioning

### Core Positioning Statement
Senior Software Engineer specializing in .NET, Azure, API integration, Angular, and enterprise platform modernization.

### Supporting Brand Headline
Enterprise .NET & Azure Engineer | Web API, Cloud Integration

### Brand Attributes
- technically credible
- modern and premium
- enterprise-oriented
- polished but not flashy
- professional and confident

## 3. Visual Direction

The portfolio should follow the same overall feel as the chosen inspiration:
- dark navy premium background
- circular profile image on the right
- left-aligned hero text
- top horizontal navigation
- minimal icon links
- single-page scrolling layout
- strong typography hierarchy
- subtle modern motion only

This is inspiration for layout composition and tone, not something to clone literally.

## 4. Key Product Objectives

The site must communicate within the first few seconds that John:
- is a real senior engineer
- works across backend, frontend, integration, and cloud concerns
- has enterprise system experience
- is strong in .NET, Angular, APIs, and Azure
- can present work cleanly and professionally

## 5. User Goals

### Technical interviewer
- quickly evaluate stack fit
- see architecture and system-level thinking
- review selected project case studies
- validate experience depth

### Recruiter / HR
- understand role fit quickly
- download the resume
- access LinkedIn and GitHub easily
- see certifications and summary strengths

### Freelance client
- understand delivery credibility
- see technical breadth and professionalism
- identify relevant project experience

## 6. Site Sections

Required sections:
- Home
- About
- Skills
- Experience
- Projects
- Certifications
- Resume
- Contact

## 7. Functional Requirements

### Required
- single-page Angular application
- sticky or persistent top navigation
- smooth scrolling to sections
- desktop-first design
- responsive enough to remain usable on smaller screens
- downloadable resume
- external links for LinkedIn and GitHub
- real profile photo
- project showcase with both public and anonymized work
- certification display
- free deployment
- recruiter-ready GitHub repo structure

### Explicitly Excluded
- blog
- pricing page
- paid hosting
- custom domain requirement
- backend server
- CMS
- database
- contact form backend

## 8. Content Strategy

The content must prioritize:
- enterprise engineering credibility
- API and platform integration experience
- clear technical strengths
- selected project impact
- concise but strong writing

The portfolio should not feel like:
- a designer portfolio
- a student site
- a template dump
- an animation-first gimmick site

## 9. Success Criteria

The product is successful when:
- the live site looks polished and professional
- the repository looks disciplined and well-documented
- a reviewer can immediately identify John's specialization
- the portfolio can be deployed entirely for free
- the site is credible enough to share in job applications
- future public projects can be added without redesigning the foundation

## 10. Risks and Constraints

### Constraints
- zero deployment cost
- static hosting only
- no public code or screenshots for private enterprise systems
- limited current public portfolio footprint

### Mitigation
- use anonymized enterprise case studies
- make the portfolio repo itself a high-quality public artifact
- keep the implementation lean and focused
- document decisions clearly in the repository
